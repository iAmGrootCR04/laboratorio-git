Dominando Git

Guía práctica para el control de versiones en proyectos web

¿Qué es el Control de Versiones?

Es un sistema que registra los cambios realizados en un archivo o conjunto de archivos a lo largo del tiempo, de modo que sea posible recuperar versiones específicas más adelante.

Git permite que varios desarrolladores trabajen en el mismo proyecto simultáneamente sin sobrescribir el trabajo de los demás mediante el uso de ramas.

Instalación y Configuración

Antes de empezar, debes configurar tu identidad en la terminal:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Estados de un Archivo

  • Modified: Cambios realizados pero no confirmados.
  • Staged: Cambios marcados para ir en la próxima captura.
  • Committed: Datos almacenados de forma segura en la base local.

Dato Curioso

Git fue creado en 2005 por Linus Torvalds, el mismo creador del kernel de Linux, tras tener problemas con el sistema que usaban anteriormente.

Comandos de Ramas (Branches)

Acción Comando
Crear nueva rama git branch nombre-rama
Cambiar de rama git checkout nombre-rama
Fusionar ramas git merge nombre-rama
Eliminar rama git branch -d nombre-rama
